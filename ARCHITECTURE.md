# Architecture

Живой документ. Только зафиксированные технические решения.

---

## Слои приложения

```
[ UI Layer        ]  рендеринг, темы, touch-взаимодействие
[ Layout Layer    ]  канвас, элементы, edit/performance mode
[ Message Layer   ]  нормализованные MIDI-сообщения
[ Transport Layer ]  WebMIDI API → порт
[ Settings        ]  выбор порта, канал по умолчанию
```

### UI Layer
Рендеринг и взаимодействие. Как нарисовать элемент, анимации, визуальный фидбек, темы. Не знает про MIDI.

### Layout Layer
Данные и логика канваса. Какие элементы существуют, где находятся, как настроены, какой режим активен. Не знает как элементы нарисованы.

### Message Layer
Нормализованный внутренний формат сообщений. Транспорт не знает про элементы, элементы не знают про транспорт.

```json
{ "type": "cc", "channel": 1, "cc": 74, "value": 127 }
{ "type": "note_on", "channel": 1, "note": 60, "velocity": 100 }
{ "type": "note_off", "channel": 1, "note": 60, "velocity": 0 }
```

### Transport Layer

Единственная реализация — WebMIDI API. WebMIDI и RTP-MIDI это один транспорт, разница только в выборе порта.

```
MIDITransport
  connect(portId)
  disconnect()
  send(message)
  getPorts() → list
  onPortsChange(callback)
```

OSC — не входит в этот проект.

## MIDI-адресация

Два независимых адресных пространства:

| Тип элемента | Протокол | Диапазон |
|---|---|---|
| Кнопки, пады | Note On/Off | Ноты 0–127 |
| Фейдеры, knobs, XY-pad | CC | Safe-zone 70–119 |

**CC 120–127 исключены** — Channel Mode Messages (All Notes Off, All Sound Off и др.) выполняются большинством девайсов без маппинга.

Один канал (канал 1 по умолчанию). В будущем при появлении страниц — каждая страница получает свой канал.

## Структура файлов

Минимальная на старт. Новый файл появляется когда существующий становится трудно читать.

```
/midi-controller
  index.html
  /core
    transport.js
    layout.js
  /design-system
    tokens.css
    elements.css
  /elements
    (появляется с первым элементом)
  /ui
    canvas.js
```

## Дизайн-система

Одна тёмная тема. CSS-переменные именуются семантически (`--bg`, `--accent`, `--surface`) — не цветовыми именами. Это позволит добавить светлую тему в будущем без правки разметки.

Два файла:
- `tokens.css` — цвета, размеры, отступы, touch targets
- `elements.css` — стили всех MIDI-элементов

---

## Режимы

**Edit mode** — настройка layout. Можно добавлять, перемещать, удалять элементы, менять параметры.

**Performance mode** — только контролы. Никаких случайных изменений layout.

Переключение одним жестом. Чёткое визуальное разделение между режимами.
