# Building Principles

Живой документ. Описывает цель проекта и принципы которыми руководствуемся.

---

## Что мы строим

Персональный кастомизируемый MIDI-контроллер для живых выступлений на iPad. Пользователь собирает свой интерфейс из элементов на канвасе и использует его на сцене вместо физического железного контроллера.

Проект личный — один разработчик и заказчик. Не претендует на продажу или маркетинг.

---

## Принцип 1 — Один элемент, одно сообщение

Приложение эмулирует физический контроллер из магазина. Пользователь мог бы иметь набор энкодеров и кнопок — и ничего больше. Каждый элемент независим и отправляет ровно одно MIDI-сообщение. Никакой логики связей между элементами, никаких зависимостей, никаких цепочек. Сложность намеренно ограничена.

> Тест: если нужно описывать логику словами "когда X, то Y" — это нарушение принципа.

---

## Принцип 2 — Простота настройки

Настройка элемента занимает секунды. Минимум параметров, никакой скриптовой логики. Всё что можно автоматизировать или скрыть — скрывается. Пользователь добавил фейдер, назначил CC, использует — всё.

---

## Принцип 3 — Физическая метафора

Каждый элемент должен быть понятен без объяснений — пользователь узнаёт фейдер, кнопку, пад. Физические инструменты — метафора, не ограничение. Экранные возможности (лейбл, подсветка, цифровое значение) используются чтобы улучшить метафору, не чтобы усложнить её.

---

## Принцип 4 — Сцена как главный сценарий

Все решения принимаются с позиции живого выступления в тёмном помещении. Интерфейс должен быть читаем, надёжен и не требовать внимания во время перформанса. Настройка происходит до выступления, не во время.

---

## Принцип 6 — Синхронизация состояния

Состояние контроллера и состояние назначения (Ableton, синтезатор) должны совпадать всегда. Это требование, не желательный эффект.

Основной механизм — **двусторонний транспорт**. Контроллер отправляет сообщение на свой адрес; Ableton применяет его к маппнутому параметру и возвращает актуальное состояние тем же MIDI-сообщением на тот же адрес. Контроллер обновляет визуал без повторной отправки. См. LOG #018.

Это работает независимо от того, кто инициирует изменение:
- пользователь тапает кнопку на контроллере → Note On → Ableton флипает состояние → Ableton шлёт обратно Note On/Off с актуальным состоянием → визуал подтверждается
- пользователь кликает Mute мышью в Ableton → Ableton шлёт Note On/Off → визуал обновляется
- автоматизация двигает фейдер → Ableton шлёт CC → хэндл едет

Абсолютные протоколы (CC) остаются естественным выбором для непрерывных значений (фейдер, knob) — там «состояние» это само число, и значение, переданное в CC, тождественно состоянию. Для бинарных элементов (toggle-кнопка) выбор между CC-абсолют и Note-флип перестал быть принципиальным, когда появился фидбек: синхронизацию теперь обеспечивает двусторонний канал, а не форма отправки. См. LOG #019.

> Правило: слушать входящий MIDI и зеркалить его в визуале — всегда. Форма исходящего сообщения выбирается по смыслу элемента, не по страху рассинхрона.

Это не отменяет Принцип 1 (один элемент — одно сообщение). Элемент по-прежнему отправляет ровно одно сообщение. Приём — это зеркало того же адреса, не новое исходящее действие и не цепочка.

Настройка на стороне Ableton для двустороннего режима: Preferences → Link/Tempo/MIDI → для нужного MIDI-порта включить галку **Remote** (на Input и на Output).

---

## Принцип 5 — Визуальный язык

**Направление:** TE / Nothing — тёмная версия. Teenage Engineering meets Nothing.tech. Плоский, высококонтрастный, утилитарный. Точечная текстура фона. Никаких теней на элементах.

**Фон:** `#0c0c0c` с точечной сеткой `rgba(255,255,255,0.07)` шаг 20px. При касании — пульс по краям экрана оранжевым градиентом.

**Элементы:**
- OFF state: `#1c1c1c`, border `rgba(255,255,255,0.1)`, border-radius 4px
- ON state: `#ff4d00` заливка — максимальный контраст, мгновенно читается
- Фейдер: белая ручка на тонком треке, fill оранжевый

**Акцентный цвет:** `#ff4d00`

**Типографика интерфейса:** Space Mono, Unbounded (лого)

**Value overlay** — при касании любого элемента в центре экрана появляется крупное значение:
- Шрифт: Bitcount Grid Single (variable), 200px, weight 900, tracking -5px
- Цвет: `rgb(255, 180, 0)`
- Blend mode: `color-dodge` на контейнере `.val-overlay`
- Transition: `opacity 0.4s ease-out` на дочерних элементах, `visibility` с задержкой 0.4s

**Светлая тема** — в roadmap, не сейчас. CSS-переменные именуются семантически.
